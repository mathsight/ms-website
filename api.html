Information for developers

Using our API

This API makes it possible for you to use our analytics data within your dashboard.

We are working with marketing agencies, toolbox providers and small to medium-sized enterprises to create an extremely insightful set of metrics.
You are not allowed to white-label our data without our specific agreement. If you plan to show our data input to a third party- please don't do so without a prior discussion of the usage rules with us.

Parameters used in our API function

1. target = this will be a target domain or page that you require the analytics data for. There are two general types:

(a) onsite : main domain url of your chosen site (e.g. http://www.mydomain.com). 
(b) offsite : domain url of an external page, or backlink, that references to your chosen site (e.g. http://www.blogcatalog.com/).

2. client_target = the specific URL you wish to target, in the case of offsite (it is unlikely that you will want to analyse the whole offsite domain, and we will not be storing the whole domain's data in any case).
This can be one or more of:
(a) In the case of target =onsite (from 1.), leave this blank
(b) In the case of target =offsite (from 1.), insert the client site URL here
(no default value)
3. offset = the starting position for the purposes of pagination of the response.
This can be any value from 0 to 'count', i.e. the number of possible items in the response.
(default value = 0)

4. count = the number of entries requested for the purposes of pagination of the response.
This can be any value from 0 to 10000.
(default value = 100)

5. mode = whether you are requesting a response for all (domain) pages or just for single page.
Possible values: 'domain', 'single'. This parameter is applicable only for onsite.
(default value = 'domain')

6. metric = the type of metric you require in the response.
Available for onsite domains and specific offiste URLs associated with a single onsite domain.
Possible values:
Basic structural and content-based feature data gathered directly from this domain's html pages, analagous to an site auditor output. StatSEO traffic quotient metric based on amalgamated page feature data and Google search 'pageview' data by URL over the last 1, 3, 6, and 12 months.
In addition to the StatSEO figure this includes:
'cluster': the cluster or page type this URL has automatically been assigned as.
'health': the health rating of this page according to the StatSEO quotient (SQ) for this cluster type. 		Highlighted significant features listed with: 
(i) a ranking according to their influence upon the statseo figure.
(ii) a count of healthy and harmful pages,
and (iii) the average SQ for each group. 		Allows you to see which URLs are potentially harmful to your chosen domain.
As part of drilldown we will provide the relevant feature value, a winning mean to aim for and the difference (delta) to the improved value.
feature statseo highlight drilldown
(metric has no default value)

7. format = defines the desired format of the response. 
Possible values: 'json', 'xml', 'csv'. 
For now, our API supports only JSON . Please send us an email to tech@mathsight.org should you require other formats. 
(default value = 'json')

8. filter = If filter is absent for a query, then ALL metrics are returned.
Filter is space delimited parameter with appropriate parameters:

 
feature feature names, e.g. : "number_of_rare_words"
highlight (none)
statseo (none)
drilldown (none)
Metric filter values
9.  order
This is valid only for metric=highlight, otherwise the parameter is ignored. Sorting is made based on the influence rank (i.e. the first number in the row).
For metric=highlight response, 4 numbers in each row are expected, where influence rank is always first one:
Possible values: 'asc', 'desc'. 

10.  type
This is valid only for 'count', otherwise the parameter is ignored. 
Possible values: 'onsite', 'offsite', 'fullsite'. The last of these three will respond with both onsite and offsite values.

Our API enables you to get the following:

1. A high level summary of the onsite and offsite health status of your chosen domain. This is based upon our StatSEO Quotient figure - a measure of attractiveness to Google's search algorithm.  


               
2. A breakdown by the most influential features affecting the bottom line; your domain's organic search performance. 

               
3. A list of the most potentially harmful URLs, according to the assessment of page feature values. 

1. Count of potentially healthy/neutral/harmful URLs

ONSITE https://api.mathsight.org/ms_api/v1/count?target=http%3A%2F%2Fwww.mydomain.com&type=onsite&token={ Your MathSight key }
OFFSITE https://api.mathsight.org/ms_api/v1/count?target=http%3A%2F%2Fwww.mydomain.com&type=offsite&token={ Your MathSight key }
FULLSITE https://api.mathsight.org/ms_api/v1/count?target=http%3A%2F%2Fwww.mydomain.com&type=fullsite&token={ Your MathSight key }
